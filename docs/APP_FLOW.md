## 2. 导航结构

### 2.1 顶部导航栏（全局）

导航栏分 **未登录** 和 **已登录** 两种状态，渲染内容不同：

> 视觉规格（色号、尺寸、字体）见 `FRONTEND_GUIDELINES.md` 6.5 节「导航栏」。

**未登录状态**（仅在 `/login` 和 `/register` 页面出现）：

```
┌──────────────────────────────────────────────────────────┐
│  [Logo/项目名]                          [登录] [注册]    │
└──────────────────────────────────────────────────────────┘
```

- 仅显示 Logo + 登录/注册两个文字链接按钮。
- 当前所在页面的按钮高亮，另一个为普通态。
- 无其他导航项。

**已登录状态**（所有受保护页面）：

```
┌──────────────────────────────────────────────────────────┐
│  [Logo/项目名]    故事库  |  历史  |  画像    [头像▾]    │
│                                          ├─ 账号设置     │
│                                          ├─ 管理后台 *   │
│                                          └─ 退出登录     │
└──────────────────────────────────────────────────────────┘
* "管理后台" 仅 admin 角色可见
```

- 左侧 Logo 点击回到 `/stories`。
- 中部三个导航项，当前页面高亮（底部下划线），其余为弱化态。
- 右侧头像下拉菜单，包含：账号设置、管理后台（仅 admin）、退出登录。

### 2.2 管理员身份说明

系统不存在"进入管理员模式"的操作。管理员是数据库中的一个角色标记（`users.role = 'admin'`），具体机制如下：

1. **首次部署**：运行一次性脚手架脚本 `scripts/seed_admin.py`，该脚本会向 `users` 表插入一条 `role='admin'` 的账号。用户名和密码在脚本中或通过命令行参数指定。
2. **日常使用**：管理员用这个账号正常登录（和普通用户走同一个登录页），系统从数据库读到 `role='admin'` 后，前端做两件事：
   - 顶部导航栏的头像下拉菜单中多渲染一项「管理后台」入口。
   - 访问 `/admin/*` 路径时不被拦截（普通用户访问会被重定向回 `/stories`）。
3. **普通用户**：注册时 `role` 固定写死为 `'player'`，没有任何前端入口或 API 可以将自己提升为 admin。
4. **后端守卫**：所有 `/api/admin/*` 接口都有 `require_admin` 依赖注入。即使有人手动拼接 URL 访问管理接口，后端会返回 `403 Forbidden`。

简言之：**谁是管理员由数据库决定，前端只做条件渲染，后端做强制鉴权。**

## 3. 普通玩家核心流程

### 3.1 登录页（`/login`）

#### 3.1.1 页面结构

> 视觉规格见 `FRONTEND_GUIDELINES.md`：布局 → 7.3 节，卡片组件 → 6.6 节，动画 → 8 节「Auth 页面」。

```
┌──────────────────────────────────────────────────────────┐
│  [Logo]                                  [登录] [注册]   │  ← 未登录顶栏
├──────────────────────────────────────────────────────────┤
│                                                          │
│              ┌────────────────────────┐                   │
│              │  标题 + 副标题          │                   │
│              │  [用户名输入框]         │                   │
│              │  [密码输入框] [👁]      │                   │
│              │  [登录按钮]             │                   │
│              │  没有账号？去注册 →      │                   │
│              └────────────────────────┘                   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

- 暗色背景，居中卡片式表单。
- 密码框右侧有眼睛图标，点击切换明文/密文。
- 底部"去注册"链接跳转 `/register`（路由跳转，非 Tab 切换）。

#### 3.1.2 交互行为

| 行为 | 说明 |
|------|------|
| Enter 键提交 | 焦点在任意输入框内按 Enter 等同点击「登录」 |
| 空字段提交 | 按钮不禁用，但提交后行内显示"请填写此字段" |
| 登录失败 | 表单卡片顶部出现红色横幅："用户名或密码错误"（不区分是哪个错） |
| 登录中 | 按钮文字替换为旋转 spinner + "登录中..."，按钮禁用 |
| 登录成功 | 整个卡片做 fadeOut（200ms），然后跳转 `/stories` |
| 重定向来源 | 从受保护页被踢到登录页时，登录成功后跳回原始目标页面 |
| 已登录访问 | 已登录用户访问 `/login` 直接重定向到 `/stories` |

#### 3.1.3 表单校验

| 字段 | 前端校验 | 后端校验 |
|------|----------|----------|
| 用户名 | 非空 | 存在性检查 |
| 密码 | 非空 | 正确性检查 |

错误展示方式：红色行内提示，出现在输入框下方，12px `danger` 色文字，配合输入框边框变为 `danger` 色。

### 3.2 注册页（`/register`）

#### 3.2.1 页面结构

> 视觉规格与登录页共用同一 Auth 卡片组件，见 `FRONTEND_GUIDELINES.md` 6.6 节。

```
┌──────────────────────────────────────────────────────────┐
│  [Logo]                                  [登录] [注册]   │
├──────────────────────────────────────────────────────────┤
│                                                          │
│              ┌────────────────────────┐                   │
│              │  标题 + 副标题          │                   │
│              │  [用户名输入框]         │                   │
│              │  [显示昵称输入框]       │                   │
│              │  [密码输入框] [👁]      │                   │
│              │  [确认密码输入框] [👁]  │                   │
│              │  [注册按钮]             │                   │
│              │  已有账号？去登录 →      │                   │
│              └────────────────────────┘                   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

- 与登录页共用同一套卡片样式。卡片高度因多出两个字段而自动增长。
- 底部"去登录"链接跳转 `/login`。

#### 3.2.2 交互行为

| 行为 | 说明 |
|------|------|
| Enter 键提交 | 焦点在最后一个输入框内按 Enter 提交 |
| 注册失败（用户名已存在） | 用户名输入框下方行内提示"该用户名已被使用" |
| 注册失败（密码不一致） | 确认密码框下方行内提示"两次密码不一致"（纯前端校验，失焦或提交时触发） |
| 注册中 | 按钮文字替换为旋转 spinner + "注册中..."，按钮禁用 |
| 注册成功 | 自动登录（无需再跳登录页），卡片 fadeOut（200ms），跳转 `/stories` |
| 已登录访问 | 直接重定向到 `/stories` |

#### 3.2.3 表单校验

| 字段 | 前端校验 | 错误文案 | 后端校验 |
|------|----------|----------|----------|
| 用户名 | 非空 + 3-20 字符 + `/^[a-zA-Z0-9_]+$/` | "3-20位，仅字母/数字/下划线" | 唯一性 |
| 显示昵称 | 非空 + 1-30 字符 | "请填写昵称" | — |
| 密码 | 非空 + ≥8 字符 | "密码至少 8 位" | — |
| 确认密码 | 非空 + 与密码一致 | "两次密码不一致" | — |

前端校验时机：**失焦时校验**（blur），提交时全量复核。用户修改内容后清除该字段的错误提示。

```
[访问 /stories/:id/new-session]
    ↓
展示：
  - 作品标题与简介
  - 模式选择：● 严谨模式（默认选中） ○ 创作模式
```

- 模式一旦选定，本会话内不可更改。
- 冒险目标为必填，至少 5 个字符。

### 3.5 游玩页（核心页面）

#### 3.5.1 页面布局

> 组件视觉规格见 `FRONTEND_GUIDELINES.md`：对话气泡 → 6.2，选项按钮 → 6.3，输入区域 → 6.4，状态面板 → 6.8，模式徽章 → 6.11，布局 → 7.2，动画 → 8 节「游玩页」。

```
┌──────────────────────────────────────────────────────────┐
│ [作品名] — 严谨模式 ●                    [状态面板 ▾]   │
├────────────────────────────────────┬─────────────────────┤
│                                    │ ┌─ 当前位置: 王城   │
│   叙事对话区域                      │ │ 当前目标: 寻找公主│
│   （滚动聊天界面）                   │ │ 关键物品: 地图    │
│                                    │ │ NPC关系:          │
│   [GM] 你来到了一片幽暗的森林...     │ │  - 老者: 友善    │
│                                    │ │  - 守卫: 中立    │
│   [你] 我查看四周有没有线索          │ └─────────────────│
│                                    │                     │
│   [GM] 你注意到树干上刻着...         │  (可折叠/展开)     │
│                                    │                     │
├────────────────────────────────────┴─────────────────────┤
│ ┌─────────┐ ┌──────────┐ ┌──────────┐                   │
│ │ 选项 A  │ │ 选项 B   │ │ 选项 C   │   或自由输入 ↓   │
│ └─────────┘ └──────────┘ └──────────┘                   │
│ ┌─────────────────────────────────────────────────────┐  │
│ │ 输入你的行动...                            [发送]  │  │
│ └─────────────────────────────────────────────────────┘  │
│                                    [⚑ 标记错误]         │
└──────────────────────────────────────────────────────────┘
```

#### 3.5.2 交互行为

- **发送消息**：点击发送或按 Enter。发送后输入框清空，禁用发送按钮直到回复完成。
- **选择选项**：点击选项按钮等同于发送该选项文本。
- **流式输出**：叙事文本逐字出现（SSE）。选项在文本完成后出现。
- **状态面板**：详见下方 3.5.3 节。
- **模式徽章**：顶部始终显示当前模式（严谨/创作）。不可点击。
- **标记错误**：点击弹出浮层，选择反馈类型（设定错误 / 不符合偏好 / 其他），提交后仅影响当前会话，标记发送给管理员审核。
- **加载态**：发送后显示"思考中..."动画，直到流式内容开始到达。
- **错误态**：API 调用失败 → 显示"生成遇到问题，请重试"+ 重试按钮。风控拦截 → 系统自动柔化重试 → 仍失败则显示"当前情节暂时无法展开，请换个方向"。

#### 3.5.3 状态面板交互

> 视觉规格（颜色、字号、Tag 样式、NPC 关系色标、刷新高亮动画）见 `FRONTEND_GUIDELINES.md` 6.8 节「状态面板」+ 8 节「游玩页动画」。

**触发方式**：顶部栏右侧「状态」按钮，点击切换展开/折叠。箭头图标随之旋转 180° 指示方向。

**展开/折叠行为**：
- 面板从右侧滑入，对话区域宽度同步收缩（flex 布局自适应），不使用遮罩层。
- 面板内标题栏的关闭箭头与顶栏按钮联动。

**展示内容**：

```
┌─────────────────────────┐
│  状态面板           [▷]  │
├─────────────────────────┤
│  📍 当前位置             │
│  王城 · 东城门广场       │
│  ─────────────────────  │
│  🎯 当前目标             │
│  寻找失踪的莉莉公主      │
│  ─────────────────────  │
│  🎒 关键物品             │
│  [地图] [钥匙] [草药×3]  │
│  ─────────────────────  │
│  👥 NPC 关系             │
│  老者·赫尔曼  🟢 友善    │
│  守卫·莱恩    🟡 中立    │
│  暗影刺客     🔴 敌对    │
└─────────────────────────┘
```

**字段数据类型**：

| 字段 | 数据类型 | 空状态文案 |
|------|----------|-----------|
| 当前位置 | 字符串 | 「未知」 |
| 当前目标 | 字符串 | 「无特定目标」 |
| 关键物品 | 字符串数组 | 「暂无物品」 |
| NPC 关系 | 对象数组 `{name, stance, note}` | 「尚未遇见 NPC」 |

**数据刷新**：
- 每轮 GM 回复完成后（SSE `state_update` 事件），面板自动刷新。变化字段有高亮提示。
- 面板折叠时静默更新，展开时显示最新值。

**交互约束**：面板内无可交互元素（除折叠按钮外），所有内容只读。

#### 3.5.4 会话结束

- 用户点击顶部「结束冒险」按钮 → 确认对话框 → 会话归档为只读。
- 关闭页面不结束会话（可从历史列表恢复查看，但 V1 不支持续玩，只能查看）。

## 4. 管理员核心流程

### 4.1 作品管理

```
[访问 /admin/stories]
    ↓
作品列表表格：
  - 标题 | 状态（未入库/入库中/已入库/入库失败） | 版本号 | 章节数 | 操作
    ↓
操作按钮：
  - [上传新作品] → 弹出上传对话框（选文件 + 填标题/简介）→ 上传 → 返回列表
  - [重新入库] → 确认对话框 → 触发入库管线 → 状态变为"入库中"
  - [回滚版本] → 确认对话框 → 切换到备份版本
  - [删除] → 确认对话框 → 删除作品及所有关联数据
  - [编辑元数据] → 跳转 /admin/stories/:id/metadata
```

### 4.2 元数据编辑

```
[访问 /admin/stories/:id/metadata]
    ↓
Tab 切换：基础信息 | 角色实体 | 角色关系 | 时间线 | 章节/场景 | 敏感段落
    ↓
各 Tab 内容：
  - 基础信息：标题、简介、标签（表单）
  - 角色实体：表格（名称 | 标准名 | 类型 | 别名 | 描述 | 操作）
  - 角色关系：表格（实体A | 实体B | 关系类型 | 描述 | 置信度 | 操作）
  - 时间线：表格（事件 | 章节 | 场景 | 排序 | 参与者 | 操作）
  - 章节/场景：表格（章节号 | 标题 | 场景数 | 摘要 | 操作）→ 展开可编辑场景切分
  - 敏感段落：表格（原文片段 | 改写版本 | 风险级别 | 操作）
    ↓
所有表格支持：新增行、编辑行（弹窗表单）、删除行
    ↓
保存 → 调用 API 更新
```

### 4.3 提示词编辑

```
[访问 /admin/prompts]
    ↓
分层展示：
  - 系统规则层（全局安全指令、角色定位）
  - 检索规则层（上下文拼装指令）
  - GM 规则层（裁定逻辑、一致性约束）
  - 风格规则层（文学性、节奏、难度控制）
    ↓

    ↓

```

## 5. 错误状态汇总

| 场景 | 错误处理 |
|------|----------|
| 未登录访问受保护页面 | 重定向到 `/login` |
| 注册用户名已存在 | 表单行内错误提示 |
| 登录凭据错误 | 表单顶部错误提示 |
| 作品上传格式不支持 | 上传对话框提示 |
| 入库管线部分失败 | 管理员收到警告列表，已成功部分可用 |
| 游玩时 API 调用失败 | 游玩页显示重试按钮 |
| 游玩时风控拦截 | 自动柔化重试 → 仍失败则提示换方向 |
| 游玩时网络断开 | 显示离线提示，恢复后可继续 |
| 评测运行失败 | 评测列表标记失败，可查看错误详情 |
| 删除正在进行的会话 | 确认对话框，确认后强制归档再删除 |
| 冒险目标输入过短 | 表单行内提示 |
| 角色卡 JSON 格式错误 | 上传时提示格式不合法 |

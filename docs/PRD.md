# PRD.md — 产品需求文档

> 项目代号：**RAG 交互式叙事研究平台**
> 版本：V1
> 最后更新：2026-03-19

---

## 1. 项目概述

### 1.1 一句话描述

基于多种 RAG 架构对比的交互式文字叙事冒险平台，用户在管理员预置的小说/故事世界中，以 GM（游戏主持人）模式进行低幻觉、高一致性的沉浸式文字冒险体验。

### 1.2 项目背景

- 本项目属于 **研究复现型全栈项目**，核心目标是在真实产品场景中对比不同 RAG 检索架构对叙事忠实性与质量的影响。
- 使用 **DeepSeek** 作为唯一文本生成模型，通过远程 Embedding API（如硅基流动 SiliconFlow）进行向量化，验证低成本模型在"低幻觉叙事"任务上的可行性。
- 首版为 **本地桌面演示**，但架构需为未来公网 Web 部署预留迁移路径。

### 1.3 核心假设

| 编号 | 假设 | 验证方式 |
|------|------|----------|
| H1 | 结构化实体/关系/时间线辅助检索能显著降低设定冲突类幻觉 | 三方案评测对比 |
| H2 | 父子块分层检索在长篇小说场景下召回质量优于朴素切块 | 检索命中率对比 |
| H3 | DeepSeek 在受约束的 GM 叙事任务中可达到可接受的忠实性 | 忠实性评分 |
| H4 | 用户画像知识库化能可感知地提升叙事个性化贴合度 | 个性化评测维度 |

## 2. 目标用户

### 2.1 管理员 / 研究者（单人超级管理员）

- **身份**：项目作者本人。
- **职责**：上传作品、维护知识库元数据、配置 RAG 实验方案、配置提示词、切换 RAG 架构、查看评测结果、查看用户会话与反馈、审计操作日志。
- **权限**：完整系统访问，包括所有用户会话的完整记录。

### 2.2 普通玩家

- **身份**：注册用户。
- **职责**：选择作品、设定模式与目标、进行交互式叙事游玩、查看历史会话、查看自己的画像摘要、提交错误反馈。
- **权限**：仅访问自己的会话与画像数据，使用公共作品知识库。

## 4. 功能详细说明

### 4.1 账号系统（F01）

- **注册字段**：用户名（唯一）、密码、显示昵称。可选：头像路径、个人简介。
- **登录方式**：用户名 + 密码，本地 JWT 令牌鉴权。
- **密码存储**：bcrypt 哈希，禁止明文存储。
- **隔离范围**：会话数据 + 用户画像按 `user_id` 严格隔离；公共作品知识库不隔离，所有用户共享。
- **角色模型**：仅 `admin`（1 人）和 `player`（多人）两种角色。
- **不做**：邮箱验证、密码重置、OAuth 第三方登录、记住登录状态。

### 4.2 作品知识库管理（F02）

#### 4.2.1 上传

- **权限**：仅管理员可上传。
- **支持格式**：`.pdf`、`.docx`、`.txt`、`.md`、结构化 `.json`（角色卡、世界设定、剧情树骨架）。
- **存储**：原始文件存文件系统 `data/uploads/`，SQLite 记录引用路径。
- **入库触发**：上传后管理员手动触发入库流程。

#### 4.2.2 自动预处理管线

按顺序执行以下步骤，整体作为一个入库任务：

1. **文档解析**：提取纯文本，尽力保留章节结构。DOCX/PDF 以 best-effort 解析，允许部分失败并警告。
2. **章节/场景切分**：自动识别章节边界与场景边界。
3. **自动切块**：在场景/段落粒度进行文本切块。
4. **实体抽取**：识别角色、地点、组织等实体，尝试归并别名为标准实体。
5. **关系抽取**：识别角色间关系类型与描述。
6. **时间线抽取**：识别关键事件及其先后顺序。
7. **摘要生成**：为章节与父块生成摘要。
8. **敏感内容文艺化改写**：识别高风险段落，生成比喻性/文艺化替代表达。保留原文归档，生成时使用改写版。
9. **向量化入库**：调用远程 Embedding API 生成向量，写入向量存储。

#### 4.2.3 入库失败处理

- 部分章节解析失败 → 已成功部分入库，失败部分标记警告，原始文本保留供人工清洗。
- 管理员可在后台手动修正章节切分、实体、关系、时间线、敏感段落改写。

#### 4.2.4 版本管理

- 同一作品重新入库时：保留最新版 + 上一版备份。
- 旧会话绑定旧版本不受影响，新会话自动使用最新版。
- 管理员可将作品回滚到备份版本（仅影响知识库，不影响已有会话）。

### 4.3 三种 RAG 检索方案（F03）

三种方案共用同一个入库管线的产物，区别在于检索阶段的策略：

#### 方案 A — 朴素混合 RAG（基线）

- 检索方式：BM25 关键词检索 + 向量语义检索，融合排序取 Top-K。
- 上下文：直接拼装检索到的原文切块。
- 适用场景：验证最基础检索能力的下限。

#### 方案 B — 父子块 / 章节-场景分层检索

- 检索方式：先用向量检索命中子块（场景/段落粒度），然后向上取父块（章节摘要 + 完整场景文本）。
- 上下文：父块摘要 + 命中子块原文。
- 优势假设：长篇小说中跨段落的上下文完整性更好。

#### 方案 C — 结构化实体 / 关系 / 时间线辅助检索

- 检索方式：先根据查询抽取涉及的实体名，从结构化表中拉取相关实体档案、关系、时间线事件；再补充向量检索命中的原文切块。
- 上下文：结构化事实（角色档案 + 关系 + 时间线） + 原文切块。
- 优势假设：对角色关系冲突、时间线矛盾类幻觉有显著抑制效果。

#### 方案切换

- 仅管理员在后台可切换当前生效的 RAG 方案。
- 普通用户不可见、不可切换。
- 切换后新会话使用新方案，旧会话保持原方案配置。

### 4.4 交互式叙事引擎（F04 + F05）

#### 4.4.1 开局流程

1. 用户选择一部作品进入。
2. 用户选择模式：严谨模式（默认）或创作模式。一旦选定，本会话内不可切换。
3. 用户输入本次冒险的目标/动机（自由文本）。
4. 系统根据作品资料检索并组织一个开场场景。

#### 4.4.2 每轮交互流程

```
用户输入（自由文本或选择选项）
       ↓
[检索阶段] 按当前 RAG 方案检索作品知识库 + 用户画像库
       ↓
[上下文拼装] 按模式动态调整优先级，压缩到 Token 预算内
       ↓
[生成阶段] DeepSeek 生成叙事回复 + 状态更新提案 + 建议选项
       ↓
[校验阶段] 程序校验状态更新提案的合理性
       ↓
[安全阶段] 文艺化/跳过高风险表达
       ↓
[输出] 流式返回叙事文本 + 渲染选项面板 + 更新状态
```

#### 4.4.3 叙事模式

| 维度 | 严谨模式（默认） | 创作模式 |
|------|------------------|----------|
| 忠实性要求 | 结论必须被原文支持 | 不违背已有设定即可 |
| 创作自由度 | 不新增设定 | 可补全细节 |
| 上下文优先级 | 作品证据 > 自动抽取 > 管理员修订 | 可适当降低证据权重 |
| 前端标识 | 显示「严谨模式」徽章 | 显示「创作模式」徽章 |

#### 4.4.4 叙事角色

系统同时支持两种叙事角色：

- **GM 裁定者**：以 TRPG 主持人身份裁定玩家行动的结果，驱动剧情推进。
- **单角色对话**：当场景聚焦于与某个 NPC 互动时，系统以该角色的口吻与玩家对话。

#### 4.4.5 选项与建议

- 每轮回复末尾附带 2-4 个建议选项（由模型动态生成，参考作品设定中的可能发展方向）。
- 用户也可忽略选项，自由输入任意行动。
- 无预设严格剧情树，系统仅从小说内容中自动提炼可能方向，效果"看起来像分支"。

#### 4.4.6 流式输出

- 首选流式逐字输出（SSE），但不是硬性要求。
- 选项面板在叙事文本完成后再渲染。

#### 4.4.7 内容安全策略

- 系统始终将成人/暴力内容做文艺化/隐喻化表达。
- 对高风险细节采用"叙事跳过"策略：只保留情节结果，不展开过程。
- 安全策略由管理员在后台配置，对用户不可见、不可调。
- 当 DeepSeek API 风控拦截时 → 自动柔化表达后重试一次 → 仍失败则退化为摘要/解释模式，提示用户"证据不足，换个方向试试"。

### 4.5 用户画像系统（F06）

#### 4.5.1 画像来源

- **行为推断**：从剧情选择与对话风格中自动总结偏好。
- **结构化导入**：用户可上传角色卡 / 人设 JSON。
- **对话推断**：从历史对话中逐步提炼。

#### 4.5.2 画像结构

- **全局画像**（跨作品通用）：阅读/叙事风格偏好、理解难度偏好、行为/价值倾向。
- **作品覆写画像**（按作品存储）：该世界中的身份/设定、NPC 关系、阶段目标。

#### 4.5.3 更新策略

- 系统全自动更新，不需要用户确认。
- 系统先判断新推断是"全局偏好"还是"作品内偏好"，再决定写入全局画像或作品覆写。
- 画像更新日志仅在后台记录，不向用户展示解释。

#### 4.5.4 用户可见性

- 用户可在画像页查看当前画像摘要/标签（只读）。
- 不可编辑、不可删除、不可关闭自动学习。
- 不可回滚到历史版本。

#### 4.5.5 画像数据保留

- 默认长期保留，管理员可删除。
- 支持按时间周期自动清理部分旧推断依据。

### 4.6 评测系统（F07）

#### 4.6.1 触发方式

- 管理员在后台手动发起评测运行。
- 管理员可抽取真实会话样本送评。

#### 4.6.2 测试用例

- 由系统根据作品内容自动生成候选测试题，管理员筛选确认。
- 题型包括：事实问答、时间线问答、续写一致性、GM 裁定、个性化贴合。

#### 4.6.3 评分维度

- **忠实性**：结论是否被原文支持 + 是否与设定矛盾 + 是否与会话状态冲突 + 敏感内容是否安全改写。
- **叙事质量**：连贯性、角色说话风格稳定、GM 规则一致、沉浸感、尊重玩家选择。

#### 4.6.4 评分方式

- 评分规则（Rubric）+ 证据片段标注，不准备标准答案。
- LLM 评委辅助打分 + 自动化指标统计。

#### 4.6.5 结果展示

- 方案级汇总分数（三方案并排对比）。
- 每道测试题可单独查看详情。
- 展示用到了哪些结构化事实。
- 仅管理员可见。

### 4.7 管理员控制台（F08）

| 模块 | 能力 |
|------|------|
| 作品管理 | 上传、删除、重建知识库、回滚版本 |
| 元数据编辑 | 作品基础信息、角色实体与别名、角色关系、时间线事件、章节/场景切分、敏感段落改写。全部使用表格 + 表单 UI。V2 再做可视化图编辑。 |
| 提示词编辑 | 分层提示词（系统规则、检索规则、GM 规则、风格规则）在线编辑 |
| RAG 方案切换 | 选择当前生效的检索方案（A/B/C） |
| 评测面板 | 发起评测、查看结果、方案对比 |
| 会话查看 | 完整对话记录 + 使用的模式与方案配置 + 用户反馈标记 |
| 审计日志 | 管理员操作记录（P2） |

### 4.8 会话状态管理

#### 4.8.1 状态模型

采用 **结构化状态对象 + 事件日志** 双层模型：

- 结构化状态：当前位置/场景、当前目标/任务、关键物品、关键 NPC 关系/立场标记。
- 事件日志：每轮交互的关键事件记录（不可变追加）。

#### 4.8.2 状态更新

- 模型提议状态更新 → 程序校验合理性后写入。
- 软约束：主要提醒模型尽量一致，不做硬性拦截。
- 管理员可在调试时手动修正会话状态。

#### 4.8.3 状态可见性

- 前端提供可折叠面板，用户可展开查看当前状态。

### 4.9 上下文拼装策略

当单轮 Token 预算不足以容纳全部信息时，按以下策略处理：

1. **先压缩再保留**：对所有上下文层尽量做摘要压缩，而非直接丢弃。
2. **按模式动态调整优先级**：
   - 严谨模式：系统规则 > 作品证据 > 会话状态 > 用户画像。
   - 创作模式：系统规则 > 会话状态 > 作品证据 > 用户画像。

## 7. 约束条件

| 维度 | 约束 |
|------|------|
| 模型 | 文本生成仅用 DeepSeek API，单模型包办所有任务（生成、抽取、评测） |
| Embedding | 远程 API，推荐硅基流动 SiliconFlow（BAAI/bge-m3），不下载本地模型 |
| 硬件 | 32GB 内存、无独立 GPU、CPU-only 本地运行 |
